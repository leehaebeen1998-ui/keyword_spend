# 저장 구조 및 인덱스 정책

## 저장 구조

다운로드 로우 파일은 브랜드 단위로 쌓습니다.

```text
save_root_path/
└─ 브랜드명/
   └─ 매체/
      └─ 실행일(YYYYMMDD)/
         └─ 일별 로우/
            └─ {media}_{account_name}_raw_{start}_{end}.{ext}
```

예:

```text
보고서 다운로드/
└─ 법무법인_태하/
   └─ Google SA/
      └─ 20260626/
         └─ 일별 로우/
            └─ google_sa_법무법인태하_01_형사_raw_20260615_20260621.csv
```

동일 파일명이 이미 있으면 기존 파일을 삭제하지 않고 `_002`, `_003`처럼 번호를 붙여 저장합니다.

## Google 분리

기존 통합 `Google Ads` 매체는 다음처럼 분리합니다.

- `Google SA`: 검색광고 보고서
- `Google DA`: 디스플레이 광고 보고서

다운로드 절차는 동일하므로 같은 Google Ads 다운로더 로직을 재사용합니다.
설정과 저장 폴더만 분리됩니다.

## 인덱스 방식 결론

현재 다운로드 단계에서는 **파일명 자동 인덱스 방식**을 사용합니다.

이유:

- 원본 로우 파일을 수정하지 않아 검증이 쉽습니다.
- 같은 기간을 여러 번 받아도 과거 파일이 보존됩니다.
- 매체 로그인/다운로드 안정성과 취합/분석 로직을 분리할 수 있습니다.

엑셀 인덱스 시트는 다운로드 단계가 아니라, 이후 통합 보고서나 성과 분석 단계에서 생성하는 편이 좋습니다.
수동 인덱스 입력 방식은 계정 수가 늘어날수록 휴먼 에러 가능성이 커서 기본 방식으로 두지 않습니다.
